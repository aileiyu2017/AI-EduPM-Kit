## 二、AI能力矩阵

### 2.1 能力清单

| AI能力 | 描述 | 优先级 | 技术方案 | 评估指标 |
|--------|------|--------|----------|----------|
| 文档解析 | 将教材/课件解析为结构化知识 | P0 | Python+OCR | 解析成功率>95% |
| 语义检索 | 根据问题召回相关知识 | P0 | Embedding+向量库 | 召回率>90% |
| 答案生成 | 基于检索内容生成回答 | P0 | LLM+Prompt工程 | 准确率>90% |
| 来源引用 | 答案附带知识来源 | P0 | 上下文追踪 | 引用完整率>95% |
| 置信度评估 | 评估答案可靠程度 | P1 | 规则+模型 | 分级准确率>85% |

### 2.2 能力边界

```
【明确AI的能力边界，这是产品规格中最重要的部分】

AI可以：
✓ 回答课程知识范围内的问题
✓ 解释概念、公式、定理
✓ 提供例题解析（知识库中有的话）
✓ 推荐相关学习资源

AI不可以：
✗ 回答超出课程范围的问题（明确拒绝）
✗ 替代教师做最终评判
✗ 生成可能有害或不当的内容
✗ 泄露其他学生的信息

灰色地带（需人工确认）：
? 涉及主观评价的问题
? 需要创造性思考的问题
? 跨学科的综合问题
```

## 三、数据流设计

### 3.1 数据流图

```
[用户提问]
    │
    ▼
[Query理解] → 意图识别 + 实体提取 + 问题改写
    │
    ▼
[检索层] → 向量检索 + 关键词检索 + 重排序
    │
    ▼
[上下文组装] → Top-K文档 + Prompt模板 + 约束条件
    │
    ▼
[生成层] → LLM推理 + 后处理 + 安全过滤
    │
    ▼
[输出层] → 结构化答案 + 来源引用 + 置信度标记
    │
    ▼
[用户反馈] → 点赞/点踩/举报 → 模型优化
```

### 3.2 关键数据定义

| 数据对象 | 定义 | 来源 | 存储 |
|----------|------|------|------|
| 知识块(Chunk) | 结构化后的知识单元 | 文档解析 | 向量数据库 |
| 对话Session | 一次完整的问答交互 | 用户行为 | 关系数据库 |
| 反馈记录 | 用户对答案的反馈 | 用户行为 | 关系数据库 |

```
【定义AI的角色和行为约束】

角色：你是{课程名称}的智能助教，基于课程资料帮助学生理解知识。

约束：

输出格式：
1. 直接回答
2. 关键要点（如有）
3. 来源引用
4. 置信度（高/中/低）
```

### 4.3 教育场景 LLM 参数推荐

| 场景 | temperature | max_tokens | top_p | presence_penalty | 说明 |
|------|-------------|------------|-------|------------------|------|
| 知识点问答 | 0.1-0.2 | 300-500 | 0.9 | 0.1 | 追求确定性，减少幻觉 |
| 概念解释 | 0.2-0.3 | 500-800 | 0.9 | 0.1 | 允许适度展开 |
| 习题辅导 | 0.1-0.2 | 800-1000 | 0.85 | 0.2 | 步骤要准确，不能跳步 |
| 创意写作/开放题 | 0.5-0.7 | 1000+ | 0.95 | 0.3 | 鼓励创造性 |
| 评估/诊断 | 0.1-0.2 | 500-800 | 0.85 | 0.2 | 结果要稳定可复现 |
| 对话/追问 | 0.3-0.4 | 300-500 | 0.9 | 0.1 | 自然但有约束 |

#### 参数调优说明

**temperature（温度）**
- 为什么不能更高：温度>0.5时，模型输出随机性显著增加，容易产生"幻觉"或编造知识点，教育场景对准确性要求高，不接受不确定的生成内容
- 为什么不能更低：温度<0.1时，模型输出趋于重复，缺乏必要的语言灵活性，尤其在开放题和对话场景会显得机械

**max_tokens（最大长度）**
- 为什么不能更多：过长的输出不仅增加响应延迟和API成本，也容易导致模型"发挥过度"，偏离核心知识点，增加幻觉风险
- 为什么不能更少：过短的输出无法完整呈现解题步骤或多维度分析，影响学习效果

**top_p（核采样）**
- 为什么不能更高：top_p接近1时等同于全开采样，小概率词汇也会被选中，可能导致事实性错误出现在回答中
- 为什么不能更低：top_p<0.8会过度限制词汇选择，使回答单调、缺乏表达多样性，尤其在创意写作场景效果差

**presence_penalty（存在惩罚）**
- 为什么不能更高：>0.5时会过度抑制重复词汇，可能导致关键术语被错误替换或遗漏，破坏概念表达的准确性
- 为什么不能更低：<0时会鼓励重复，导致回答啰嗦、信息密度低，影响阅读体验

## 五、Prompt 模板库

> 以下模板可直接复制使用，替换 `{占位符}` 为实际内容即可。

### 模板1：教育AI系统角色设定

```
你是{课程名称}的智能助教，基于课程资料帮助学生理解知识。

【核心约束】
- 只使用提供的课程资料回答，不引入外部知识
- 如果资料不足以回答，明确告知"根据现有资料，无法完整回答"
- 回答必须附带来源引用
- 超出课程范围的问题明确拒绝

【回答格式】
1. 直接回答（2-3句话）
2. 关键要点（如有步骤/要点，用列表）
3. 来源引用 [文档名 第X章第Y节]
4. 置信度：高/中/低

【语气要求】
- 鼓励性、耐心、不使用过于学术化的语言
- 对不确定的内容诚实说明
```

### 模板2：教学质量诊断报告生成

```
基于以下多源数据，生成教学质量诊断报告。

【输入数据】
- 教学视频分析结果：{视频分析结果}
- 学生评教数据：{评教数据}
- 督导记录：{督导记录}
- 成绩数据：{成绩数据}

【输出要求】
- 诊断维度：教学行为、课堂互动、内容结构、教学效果
- 每个维度评分（1-5分）+ 证据片段 + 改进建议
- 整体诊断摘要（100字以内）
- 不评判教师个人，只关注教学改进
```

### 模板3：RAG 问答上下文组装

```
【角色】{课程名称}智能助教
【资料】{检索到的Top-K文档片段}
【问题】{用户问题}

请基于以上资料回答，遵循：
1. 如果资料中没有相关信息，回答"这不在我的知识范围内"
2. 引用资料时标注来源
3. 温度参数：0.2（低随机性，高确定性）
4. 最大长度：500字
```

### 模板4：AI 回答质量自检

```
请评估以下AI回答的质量，从5个维度打分（1-5分）：

【AI回答】{回答内容}
【用户问题】{原始问题}
【参考资料】{参考内容}

评估维度：
1. 准确性：回答是否与参考资料一致
2. 完整性：是否回答了问题的所有方面
3. 可解释性：是否附带了来源引用
4. 安全性：是否包含可能有害的内容
5. 边界控制：是否回答了超出范围的内容

输出格式：各维度分数 + 总分 + 改进建议
```

### 模板5：学生风险预警分级

```
基于以下学生数据，判断风险等级并给出干预建议。

【学生数据】
- 出勤率：{出勤率}
- 作业提交率：{作业提交率}
- 近期测试成绩：{成绩}
- 课堂互动频次：{互动频次}

【分级标准】
🟢 正常：所有指标正常
🟡 关注：1-2项指标异常
🔴 高风险：3项以上指标异常或单项严重异常

【输出要求】
- 风险等级 + 判定依据
- 差异化干预建议（具体可执行）
- 是否需要人工介入
```

## 六、评估体系

### 6.1 技术指标

| 指标 | 定义 | 计算方法 | 目标 |
|------|------|----------|------|
| 召回率 | 检索到相关文档的比例 | 人工标注对比 | >90% |
| 准确率 | 答案正确的比例 | 人工抽样评估 | >90% |
| 幻觉率 | 包含无法验证信息的比例 | 对比知识库 | <5% |
| 延迟 | 从提问到答案的时间 | 监控日志 | <2秒 |

### 6.2 业务指标

| 指标 | 定义 | 采集方式 | 目标 |
|------|------|----------|------|
| 可回答率 | 成功回答的问题比例 | 系统统计 | >85% |
| 用户满意度 | 用户对答案的满意度 | 反馈按钮 | >4.0/5 |
| 重复提问率 | 同一问题多次提问的比例 | 行为分析 | <10% |
| 人工转接率 | 转人工处理的比例 | 系统统计 | <15% |

## 七、安全与合规

### 7.1 内容安全

| 风险类型 | 防控措施 | 责任方 |
|----------|----------|--------|
| 幻觉/错误信息 | RAG约束+人工审核 | 产品+技术 |
