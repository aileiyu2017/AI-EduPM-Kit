# PRD模板：教育AI产品需求文档

> 专为教育AI场景设计的产品需求文档模板。

---

## 一、产品概述

### 1.1 背景

```
【描述业务背景和教育场景痛点】
例如：当前高校教学质量评估高度依赖人工督导，覆盖率不足30%，
评估标准不统一，反馈滞后。
```

### 1.2 目标

```
【产品目标，遵循SMART原则】
- 目标1：3个月内将教学评估覆盖率从30%提升至60%
- 目标2：标准化评估维度，减少督导间评分差异
- 目标3：评估反馈周期从学期末缩短至2周内
```

### 1.3 范围

```
【定义产品边界——做什么、不做什么】

In Scope:
- 教学视频自动分析
- 多源数据融合诊断
- 诊断报告生成与分发

Out of Scope:
- 不涉及教师绩效考核
- 不替代人工督导，仅辅助
- 不处理非教学类视频
```

## 二、用户分析

### 2.1 用户画像

| 角色 | 描述 | 核心诉求 | 使用频率 |
|------|------|----------|----------|
| 教学督导 | 负责听课评估教学质量 | 高效完成评估任务 | 每周 |
| 教师 | 被评估对象 | 获得有价值的改进建议 | 每学期 |
| 教务管理者 | 管理教学质量 | 宏观掌握教学质量状况 | 每月 |

### 2.2 用户旅程

```
【描述用户使用产品的完整流程】

教学督导视角：
1. 登录系统 → 查看本周推荐听课清单
2. 查看AI预分析报告 → 带着重点进课堂
3. 听课 → 记录人工观察
4. 课后 → 对比AI分析与人工观察
5. 确认/修改诊断结论
6. 生成最终报告 → 反馈给教师
```

## 三、AI产品机制设计

### 3.1 AI能力定义

```
【明确AI在这个产品中能做什么、不能做什么】

AI能：
- 分析教学视频，提取行为特征
- 融合多源数据生成初步诊断
- 推荐高价值听课对象

AI不能：
- 做最终评估结论（需人工确认）
- 评判教师教学水平好坏
- 替代督导的专业判断
```

### 3.2 人机协同模式

```
【选择对应的人机协同模式，详见 docs/human-ai-patterns.md】

主模式：模式1 - AI辅助决策
辅模式：模式3 - AI引导发现、模式5 - AI预测预警

协同流程：
[用户] 提交教学视频
  → [AI] 自动分析生成初步报告
  → [人] 督导审核、补充、确认
  → [系统] 生成最终报告
  → [AI] 收集反馈优化模型
```

### 3.3 可控性设计

```
【描述用户如何控制AI的行为】

- 督导可以修改AI的任何诊断结论
- 可以调整AI分析的敏感度和严格度
- 可以选择关闭某些自动分析维度
- 所有修改记录可追溯
```

### 3.4 可解释性设计

```
【描述AI决策如何被理解和追溯】

- 每条诊断结论附带视频证据片段
- 综合评分可拆解为各维度贡献
- 提供"为什么这样诊断"的说明
- 不同角色看到不同粒度的解释
```

## 四、功能需求

### 4.1 功能清单

| 优先级 | 功能模块 | 功能描述 | 验收标准 |
|--------|----------|----------|----------|
| P0 | 视频分析 | 自动分析教学视频 | 支持10种+教学行为识别 |
| P0 | 诊断报告 | 生成结构化诊断报告 | 包含维度评分+证据+建议 |
| P0 | 人工审核 | 督导审核修改AI结论 | 修改率<30%为健康指标 |
| P1 | 对象推荐 | 推荐高价值听课对象 | 推荐准确率>80% |
| P1 | 趋势分析 | 教学质量趋势追踪 | 支持学期/年度对比 |
| P2 | 预警通知 | 异常自动预警 | 误报率<15% |

### 4.2 核心流程

```
【用流程图或步骤描述核心业务流程】

诊断流程：
1. 视频上传 → 格式校验 → 进入分析队列
2. AI分析（耗时10-30分钟）
   - 视频理解（行为/互动/内容）
   - 数据融合（视频+评教+成绩）
   - 诊断生成（多维度评分）
3. 人工审核
   - 督导查看AI报告
   - 对比人工观察
   - 确认或修改结论
4. 报告分发
   - 生成最终报告
   - 推送给教师（仅改进建议，不含评分）
   - 数据汇总至管理者视图
```

## 六、数据需求

### 6.1 数据源

| 数据类型 | 来源 | 用途 | 敏感度 |
|----------|------|------|--------|
| 教学视频 | 录播系统 | AI分析 | 高 |
| 学生评教 | 教务系统 | 融合分析 | 中 |
| 成绩数据 | 教务系统 | 效果验证 | 高 |
| 督导记录 | 本系统 | 反馈优化 | 中 |

### 6.2 数据安全

```
【描述数据保护措施】

- 视频存储：加密存储，访问需授权
- 数据传输：TLS加密
- 数据使用：最小化原则，仅用于教学改进
- 数据保留：学期结束后脱敏归档
- 数据删除：支持教师申请删除个人数据
```

## 七、非功能需求

### 7.1 性能

| 指标 | 要求 | 说明 |
|------|------|------|
| 视频分析速度 | 1小时视频<30分钟 | 异步处理 |
| 报告生成延迟 | <5秒 | 同步操作 |
| 并发支持 | 100路视频同时分析 | 开学季峰值 |
| 系统可用性 | 99.5% | 教学周期间 |

### 7.2 兼容性

- 支持主流视频格式（MP4/AVI/MOV）
- 支持主流浏览器（Chrome/Firefox/Edge/Safari）
- 支持移动端查看报告
