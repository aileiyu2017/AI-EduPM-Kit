# 教育AI产品岗面试准备清单

---

## 1. 自我介绍模板（1-2分钟）

### 结构公式
背景（学历/岗位）→ AI产品经验（年限/方向）→ 核心成果（数据量化）→ 为什么这个岗位（业务匹配度）

### 示例（LeiYu版）
> "你好，我是LeiYu，3年AI产品经理经验，专注教育场景。
> 上一家公司负责AI教学质量诊断系统，覆盖200+学校，帮助教师课后复盘效率提升40%。
> 核心成果是将视频分析准确率从72%提升至91%，并建立了人机协同的审核机制。
> 我对贵公司AI+教育的方向非常认同，尤其是大模型在个性化学习中的应用，
> 我希望把我在AI效果评估和场景落地方面的经验带过来，和团队一起打造真正好用的教育AI产品。"

### 准备建议
- 时长控制在90-120秒，超时会被面试官打断
- 成果必须有数字支撑（用户数、准确率提升、效率提升百分比等）
- "为什么这个岗位"一定要提前研究公司业务，避免泛泛而谈
- 准备1个精简版（30秒）和1个完整版（2分钟），视面试官节奏灵活调整

## 2. 项目讲述框架：STAR + AI维度

### 基础框架
| 环节 | 说明 |
|------|------|
| S-Situation | 项目背景和业务目标 |
| T-Task | 你负责的具体任务 |
| A-Action | 你采取的关键行动 |
| R-Result | 量化结果和影响 |

### AI产品特有维度
| 维度 | 提问自检 |
|------|---------|
| **A-AI机制** | 用了什么AI能力？为什么选这个方案而非其他？ |
| **H-Human-AI** | 人机如何协同？人工介入点在哪？为什么这样设计？ |
| **E-Evaluation** | 怎么评估AI效果？指标是什么？如何持续监控？ |
| **T-Trade-off** | 关键决策的取舍？为什么选A不选B？ |

### 完整示例：教学质量诊断项目
> **S**：我们在200所K12学校推广双师课堂，发现课后教学质量评估依赖人工听课，
> 每学期只能覆盖5%的课程，反馈周期2周以上。
>
> **T**：我负责设计AI教学质量诊断系统，目标是实现全量课程自动分析，
> 2小时内生成诊断报告，覆盖教学设计、师生互动、目标达成三个维度。
>
> **A**：选用多模态分析方案——语音转文字分析师生对话比例，
> 视觉分析教师走位和板书行为。模型选用轻量化方案（ResNet+Whisper-tiny），
> 部署在学校本地服务器，保障数据安全。
>
> **R**：覆盖率达100%课程，诊断报告生成时间从2周缩短至90分钟，
> 教师满意度87%，教研员使用率达95%。
>
> **A-AI机制**：选择离线分析而非实时，因为教育场景对时效要求不高，
> 离线分析成本降低80%，且可用更强模型。选用轻量模型是为了适配学校硬件。
>
> **H-Human-AI**：AI生成初稿报告后，教研员审核修正关键环节，
> 尤其关注AI标注的"疑似低质量"片段。设置置信度分级，低于阈值必审，
> 高于阈值抽查。人工审核意见回流训练集，模型每月迭代。
>
> **E-Evaluation**：准确率（对比人工标注）达91%；召回率85%。
> 业务指标：教师采纳建议率73%、报告打开率92%。每周抽查10份报告校准。
>
> **T-Trade-off**：放弃通用大模型方案，选择专用小模型。
> 大模型效果略好但推理成本高10倍、本地部署困难；小模型效果够用、
> 可控性强、维护简单。教育场景优先可解释性和成本。

## 3. 常见问题与回答思路（10个）

**Q1：你如何定义一个AI产品的成功？**
→ 不是准确率，是业务指标+用户满意度+可控性。
准确率高但没人用是失败。关键看：业务目标达成度、用户真实使用率、
AI出错时是否可控、人机协作是否顺畅。

**Q2：AI出错了怎么办？**
→ 三层防护：①兜底机制（默认安全回复/人工转接）；
②置信度分级（低置信自动转人工）；③人工审核+持续优化。
核心认知：不追求零错误，追求错误可控、可发现、可修复。

**Q3：如何平衡AI自动化和人工审核？**
→ 分场景决策：高风险必审（如学生评价、成绩判定），
中风险抽查（如内容推荐），低风险自动（如学习路径调整）。
逐步放权原则：上线初期100%审核，数据积累后逐步降低比例。

**Q4：你怎么评估AI的公平性？**
→ ①训练数据分布检查，确保覆盖不同地区/学段/性别；
②分群效果对比，识别表现差异大的子群体；
③偏见检测工具定期扫描；④建立反馈渠道，收集歧视或偏见案例。

**Q5：大模型hallucination怎么解决？**
→ 四层策略：①RAG约束，限定知识来源；②证据绑定，
回答必须引用原文/数据源；③置信度分级，不确定时主动降权或拒答；
④兜底回复，超出范围时给出标准话术，不编造。

**Q6：AI产品需求变更频繁怎么办？**
→ 建立模型版本管理机制：需求冻结期、A/B测试验证、灰度发布策略。
区分模型层变更（需重训，周期长）和策略层变更（Prompt调整，快速上线）。
需求入池前评估对模型效果的影响。

**Q7：如何向非技术团队解释AI能力边界？**
→ 用类比："AI像一个有知识的实习生，能做大量基础工作，
但需要主管审核，也会犯错。" 明确列出能做什么、不能做什么，
各给3个具体例子。避免技术术语，用业务语言。

**Q8：你做过的最复杂的AI产品决策是什么？**
→ 准备1-2个有真实trade-off的案例，重点讲决策逻辑而非技术细节。
用"当时面临A和B两个选择，我选了X，因为...，代价是...，结果证明..."
的结构。

**Q9：AI产品上线前要做哪些准备？**
→ 五维检查：①安全测试（内容安全、数据隐私）；
②效果验证（准确率、召回率、分群表现）；③合规审查（教育政策、AI法规）；
④降级方案（模型失效时如何兜底）；⑤监控告警（关键指标实时看板）。

**Q10：你对我们公司的AI产品有什么建议？**
→ 提前研究公司产品线，给1-2个具体建议。
建议框架："我注意到XX场景，如果引入XX能力，可能会带来XX效果，
当然这需要验证XX假设。" 避免泛泛而谈。

## 4. 反问环节准备

建议向面试官提出以下问题，展示对AI产品的思考深度：

1. **关于AI投入产出**："团队目前衡量AI功能ROI的核心指标是什么？
   比如教研效率提升、学生成绩变化，还是用户使用率？"

2. **关于人机协作模式**："当前AI产品的审核机制是怎样的？
   产品团队如何与算法、运营团队分工定义自动化边界？"

3. **关于技术债务**："团队在推进AI产品时，是否遇到过模型效果与产品体验冲突的情况？
   一般怎么做取舍？"

4. **关于数据与合规**："教育场景的数据合规要求很高，
   团队在训练数据和模型输出安全方面做了哪些保障措施？"

5. **关于发展方向**："公司未来1-2年在AI+教育方向的重点投入领域是什么？
   大模型应用目前处于什么阶段，有哪些已验证落地的场景？"

### 反问避坑指南
- 不问"我什么时候有晋升机会"（过于功利）
- 不问"加班多不多"（可私下了解）
- 不问官网上已明确的信息（显得没做功课）
- 不问与岗位无关的问题（聚焦AI产品）

## 5. 技术概念速查

| 概念 | 一句话解释 |
|------|-----------|
| **RAG** | 检索增强生成，AI回答前先查资料库，限定回答来源，减少胡说八道 |
| **Fine-tuning** | 微调，用领域数据训练已有大模型，让它更懂特定任务 |
| **Prompt Engineering** | 提示词工程，通过优化输入指令，引导AI输出更精准的结果 |
| **Hallucination** | 幻觉，AI生成看似合理但实际错误的内容，是教育AI必须控制的 |
| **Embedding** | 嵌入，把文字/图片转成数值向量，让计算机能计算语义相似度 |
| **A/B测试** | 对照实验，同时上线两个版本对比效果，用数据决策而非感觉 |
| **灰度发布** | 先让少量用户体验新功能，验证稳定后再全量推广 |
| **置信度** | AI对自己答案的确定程度，低置信度时应触发人工审核或兜底 |
| **多模态** | 同时处理文本、语音、图像等多种信息类型，如视频课分析 |
| **模型版本管理** | 像管理代码版本一样管理AI模型，确保可回滚、可对比、可追溯 |

## 6. 不同面试官的应对策略

| 面试官角色 | 关注点 | 应对策略 |
|-----------|--------|---------|
| HR/招聘经理 | 职业稳定性、价值观匹配 | 强调对教育AI的热情和长期规划 |
| 产品总监 | 产品方法论、决策逻辑 | 多用STAR框架，展示数据思维和用户洞察 |
| 算法/技术负责人 | 技术理解深度、可行性判断 | 准确使用术语，承认不懂的地方，展示协作意识 |
| 业务/教研负责人 | 教育场景理解、业务价值 | 用教研语言交流，强调对教学流程的理解 |

**注意**：遇到不懂的技术问题，诚实回答"这个我不确定，但我理解它的作用是...，
如果我需要深入了解，我会通过...方式学习"，比硬编更专业。

## 7. 面试前Checklist

### 提前1-2天
- [ ] 研究公司官网，了解产品线和AI应用场景
- [ ] 查看公司是否有公开的AI产品/技术博客
- [ ] 准备1-2个针对该公司AI产品的具体建议
- [ ] 复习自己简历中的每个项目，确保能讲出STAR+AHET

### 提前1小时
- [ ] 测试设备和网络（线上面试）
- [ ] 准备纸笔，方便记录面试官问题和反馈
- [ ] 深呼吸，调整心态——面试是双向选择

### 心态提醒
- 不会的问题大方承认，但给出思考方向
- 强调"我会怎么查"比硬编答案更专业
- 每个回答尽量控制在1-2分钟，观察面试官反应调整
