## 二、AI能力矩阵

### 2.1 能力清单

| AI能力 | 描述 | 优先级 | 技术方案 | 评估指标 |
|--------|------|--------|----------|----------|
| 文档解析 | 将教材/课件解析为结构化知识 | P0 | Python+OCR | 解析成功率>95% |
| 语义检索 | 根据问题召回相关知识 | P0 | Embedding+向量库 | 召回率>90% |
| 答案生成 | 基于检索内容生成回答 | P0 | LLM+Prompt工程 | 准确率>90% |
| 来源引用 | 答案附带知识来源 | P0 | 上下文追踪 | 引用完整率>95% |
| 置信度评估 | 评估答案可靠程度 | P1 | 规则+模型 | 分级准确率>85% |

### 2.2 能力边界

```
【明确AI的能力边界，这是产品规格中最重要的部分】

AI可以：
✓ 回答课程知识范围内的问题
✓ 解释概念、公式、定理
✓ 提供例题解析（知识库中有的话）
✓ 推荐相关学习资源

AI不可以：
✗ 回答超出课程范围的问题（明确拒绝）
✗ 替代教师做最终评判
✗ 生成可能有害或不当的内容
✗ 泄露其他学生的信息

灰色地带（需人工确认）：
? 涉及主观评价的问题
? 需要创造性思考的问题
? 跨学科的综合问题
```

## 三、数据流设计

### 3.1 数据流图

```
[用户提问]
    │
    ▼
[Query理解] → 意图识别 + 实体提取 + 问题改写
    │
    ▼
[检索层] → 向量检索 + 关键词检索 + 重排序
    │
    ▼
[上下文组装] → Top-K文档 + Prompt模板 + 约束条件
    │
    ▼
[生成层] → LLM推理 + 后处理 + 安全过滤
    │
    ▼
[输出层] → 结构化答案 + 来源引用 + 置信度标记
    │
    ▼
[用户反馈] → 点赞/点踩/举报 → 模型优化
```

### 3.2 关键数据定义

| 数据对象 | 定义 | 来源 | 存储 |
|----------|------|------|------|
| 知识块(Chunk) | 结构化后的知识单元 | 文档解析 | 向量数据库 |
| 对话Session | 一次完整的问答交互 | 用户行为 | 关系数据库 |
| 反馈记录 | 用户对答案的反馈 | 用户行为 | 关系数据库 |

## 四、Prompt规范

### 4.1 系统Prompt

```
【定义AI的角色和行为约束】

角色：你是{课程名称}的智能助教，基于课程资料帮助学生理解知识。

约束：
1. 只使用提供的课程资料回答
2. 如果资料不足以回答，明确告知
3. 回答必须附带来源引用
4. 使用中文回答，结构化输出
5. 对不确定的内容标注置信度

输出格式：
1. 直接回答
2. 关键要点（如有）
3. 来源引用
4. 置信度（高/中/低）
```

### 4.2 上下文模板

```
【定义检索结果如何组装到Prompt中】

资料：
---
[文档1] 来源：{文件名} 第{页码}页
{文档内容}

[文档2] 来源：{文件名} 第{页码}页
{文档内容}
---

问题：{用户问题}

请基于以上资料回答。
```

### 4.3 温度与参数

| 参数 | 值 | 理由 |
|------|-----|------|
| temperature | 0.1-0.3 | 教育场景需要稳定、确定性输出 |
| max_tokens | 500-1000 | 控制回答长度 |
| top_p | 0.9 | 限制采样空间 |

## 五、评估体系

### 5.1 技术指标

| 指标 | 定义 | 计算方法 | 目标 |
|------|------|----------|------|
| 召回率 | 检索到相关文档的比例 | 人工标注对比 | >90% |
| 准确率 | 答案正确的比例 | 人工抽样评估 | >90% |
| 幻觉率 | 包含无法验证信息的比例 | 对比知识库 | <5% |
| 延迟 | 从提问到答案的时间 | 监控日志 | <2秒 |

### 5.2 业务指标

| 指标 | 定义 | 采集方式 | 目标 |
|------|------|----------|------|
| 可回答率 | 成功回答的问题比例 | 系统统计 | >85% |
| 用户满意度 | 用户对答案的满意度 | 反馈按钮 | >4.0/5 |
| 重复提问率 | 同一问题多次提问的比例 | 行为分析 | <10% |
| 人工转接率 | 转人工处理的比例 | 系统统计 | <15% |

## 六、安全与合规

### 6.1 内容安全

| 风险类型 | 防控措施 | 责任方 |
|----------|----------|--------|
| 幻觉/错误信息 | RAG约束+人工审核 | 产品+技术 |
| 敏感内容 | 过滤模型+敏感词库 | 技术 |
| 版权内容 | 只引用课程授权资料 | 产品 |
| 隐私泄露 | 数据脱敏+权限控制 | 技术 |

### 6.2 合规要求

- [ ] 符合《个人信息保护法》
- [ ] 获得学校/学生的数据使用授权
- [ ] 数据存储在国内
- [ ] 有数据删除机制

## 七、接口定义（可选）

### 7.1 问答接口

```
POST /api/v1/ask

Request:
{
  "session_id": "string",      // 对话会话ID
  "question": "string",        // 用户问题
  "course_id": "string",       // 课程ID（用于过滤知识库）
  "context": {                 // 上下文（多轮对话）
    "previous_qa": []
  }
}

Response:
{
  "answer": "string",          // 结构化答案
  "sources": [                 // 来源引用
    {
      "document": "string",
      "page": "number",
      "relevance": "number"
    }
  ],
  "confidence": "high|medium|low",
  "processing_time": "number"
}
```
