# 贡献指南

感谢你对 EduPM-Kit 的兴趣！以下是参与贡献的方式。

---

## 贡献类型

### 1. 提交案例

如果你在教育AI领域有实践经验，欢迎提交案例：

- 参考现有案例格式（见 examples/ 目录）
- 必须包含：背景、方案设计、量化成果、经验总结
- 可以是成功或失败的经验，两者都有价值

### 2. 完善框架

评估框架和设计模式需要持续迭代：

- 发现遗漏的评估维度？提交Issue或PR
- 有新的设计模式？提交PR
- 现有内容有错误？提交Issue

### 3. 改进模板

PRD模板和产品规格书模板需要根据实践不断优化：

- 缺少必要的章节？
- 某些部分不够实用？
- 有新的模板需求？

### 4. 翻译与传播

- 将文档翻译为其他语言
- 在社交媒体/社区分享
- 在演讲/培训中引用（保留署名即可）

---

## 提交规范

### Issue 规范

- 标题清晰：[类型] 简要描述
- 类型包括：案例提交、框架完善、模板改进、错误修正、建议
- 正文说明背景、具体内容、期望结果

### PR 规范

1. Fork 本仓库
2. 创建分支：`feature/你的功能` 或 `fix/问题描述`
3. 提交修改，commit message 清晰
4. 提交 PR，说明修改内容和理由

### 内容标准

- 所有内容基于真实经验或可靠来源
- 量化数据优先（"提升15%"优于"显著提升"）
- 方法论可复用（读者能照着做）
- 中文为主，专业术语保留英文

---

## 社区行为准则

- 尊重不同观点和经验背景
- 建设性讨论，避免人身攻击
- 以教育价值和产品实践为核心
