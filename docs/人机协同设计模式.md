# 人机协同设计模式

> 教育场景中6种典型的人机协作模式，以及选型指南。

---

## 核心原则

教育AI不是替代人，而是增强人。设计人机协同系统的第一性原理：

> **AI做AI擅长的（规模化、一致性、 tirelessly），人做人擅长的（判断力、同理心、创造力）。**

---

## 6种设计模式

### 模式1：AI辅助决策（AI-Augmented Decision）

**定义**：AI生成建议/分析 → 人工审核确认 → 系统执行

**适用场景**：
- 教学质量评估
- 学生风险预警
- 资源分配建议

**典型案例：教学质量诊断**

```
教学视频 + 评教数据 + 督导记录
        ↓
    AI分析诊断
        ↓
┌──────────────────┐
│ AI输出：          │
│ - 诊断报告        │
│ - 问题维度拆解    │
│ - 证据片段        │
│ - 改进建议        │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 督导审核：        │
│ ✓ 确认/修改诊断  │
│ ✓ 补充人工观察   │
│ ✓ 调整建议优先级 │
└────────┬─────────┘
         ↓
    生成最终报告 → 反馈给教师
```

**设计要点**：
- AI结论必须有证据支撑
- 人工审核环节不可跳过
- 审核过程本身产生新数据（反哺AI）
- 记录"AI建议 vs 人工修改"的差异（持续优化）

**关键指标**：
- AI建议采纳率（目标70-85%，过高说明AI太强或人太懒）
- 人工修改的主要方向（指导AI优化）
- 决策周期（AI+人 vs 纯人工）

---

### 模式2：AI增强执行（AI-Enhanced Execution）

**定义**：AI处理重复性/规模化的执行任务 → 人工聚焦核心判断和策略

**适用场景**：
- 作业批改（客观题/初步评分）
- 课程资料结构化
- 大规模数据分析

**典型案例：课程资料结构化**

```
1000份教学文档（PDF/Word/PPT）
        ↓
┌──────────────────┐
│ AI自动处理：      │
│ - 文档解析        │
│ - 内容分块        │
│ - 标签标注        │
│ - 质量初筛        │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 教师审核：        │
│ ✓ 抽样验证质量   │
│ ✓ 修正分类错误   │
│ ✓ 补充教学注释   │
│ ✓ 确认关键内容   │
└────────┬─────────┘
         ↓
    进入知识库 → 支撑AI问答
```

**设计要点**：
- AI的准确率必须达到可接受阈值（通常>90%）
- 人工审核工作量应显著低于纯人工（目标节省70%+时间）
- 错误样本自动收集，用于模型迭代

---

### 模式3：AI引导发现（AI-Guided Discovery）

**定义**：AI提出问题线索/关联关系 → 人工深入探究和决策

**适用场景**：
- 教学问题诊断
- 学生学习路径分析
- 教育资源优化

**典型案例：教学问题定位**

```
课堂视频 + 学生成绩 + 出勤数据
        ↓
    AI关联分析
        ↓
┌──────────────────────────────┐
│ AI发现线索：                  │
│ ⚠ "该班级第三章测试得分      │
│   显著低于其他班级（p<0.05）" │
│ ⚠ "与该教师其他班级对比，     │
│   该章节教学效果偏离度+23%"   │
└──────────────┬───────────────┘
               ↓
┌──────────────────────────────┐
│ 督导/教研组长深入探究：       │
│ - 查看该章节教学视频          │
│ - 与教师沟通                  │
│ - 对比教材版本差异            │
│ - 制定针对性改进方案          │
└──────────────────────────────┘
```

**设计要点**：
- AI只做"提示"，不做"结论"
- 线索必须有统计支撑（避免AI过度联想）
- 人工探究的过程和结论记录，形成知识沉淀

---

### 模式4：AI实时反馈（AI Real-time Feedback）

**定义**：AI即时响应用户的交互 → 人工长期跟踪效果

**适用场景**：
- AI问答助手
- 智能练习推荐
- 课堂实时互动

**典型案例：AI数字教师问答**

```
学生提问："为什么光合作用需要叶绿素？"
        ↓
┌──────────────────────────────┐
│ AI实时处理：                  │
│ 1. 检索课程知识库            │
│ 2. 生成结构化答案            │
│ 3. 附带来源引用              │
│ 4. 标记置信度                │
└──────────────┬───────────────┘
               ↓
答案 + 引用 + 置信度 → 学生
               ↓
┌──────────────────────────────┐
│ 教师长期跟踪：                │
│ - 高频问题分析               │
│ - 学生满意度趋势             │
│ - 知识库覆盖缺口             │
│ - AI回答质量变化             │
└──────────────────────────────┘
```

**设计要点**：
- 响应速度必须快（<1秒）
- 实时交互的质量把控比离线更严格
- 教师端需要有"监控面板"查看AI交互数据

---

### 模式5：AI预测预警（AI Predictive Alert）

**定义**：AI基于历史数据预测风险 → 人工干预处置

**适用场景**：
- 学习风险预警（辍学/挂科）
- 教学质量下滑预警
- 资源需求预测

**典型案例：教学质量风险预警**

```
多源数据（视频/评教/成绩/出勤）
        ↓
    AI预测模型
        ↓
┌──────────────────────────────┐
│ 三级预警：                    │
│ 🟢 绿色 - 正常               │
│ 🟡 黄色 - 关注（预警准确率85%）│
│ 🔴 红色 - 高风险（需立即干预） │
└──────────────┬───────────────┘
               ↓
┌──────────────────────────────┐
│ 人工干预：                    │
│ 黄色 → 增加听课频次          │
│ 红色 → 立即约谈/教学督导介入 │
│ 干预结果记录 → 反馈优化模型   │
└──────────────────────────────┘
```

**设计要点**：
- 预警必须分级（避免狼来了效应）
- 每级预警配套明确的干预SOP
- 误报率必须控制（目标<15%）
- 预警到人（明确责任人）

---

### 模式6：AI知识检索（AI Knowledge Retrieval）

**定义**：AI负责信息召回和初步整理 → 人工综合判断和创造

**适用场景**：
- 教学资料检索
- 政策文件查询
- 跨学科知识关联

**典型案例：教学政策查询**

```
教师提问："新课程标准对高三数学的要求是什么？"
        ↓
┌──────────────────────────────┐
│ AI检索处理：                  │
│ 1. 召回相关政策文件          │
│ 2. 提取关键条款              │
│ 3. 结构化呈现                │
│ 4. 关联历史版本对比          │
└──────────────┬───────────────┘
               ↓
结构化政策摘要 + 来源链接 + 关联文件
               ↓
┌──────────────────────────────┐
│ 教师综合判断：                │
│ - 理解政策要求               │
│ - 结合自身教学实际           │
│ - 制定教学调整方案           │
│ - 创造新的教学方法           │
└──────────────────────────────┘
```

**设计要点**：
- 召回率优先（宁可多给不要漏掉）
- 信息必须结构化（方便人工快速消化）
- 来源必须可追溯

## 模式选型指南

### 决策树

```
你的教育AI产品的核心价值是什么？
│
├─→ 帮助人做更好的决策？
│   └─→ 模式1：AI辅助决策
│
├─→ 替人做重复性工作？
│   └─→ 模式2：AI增强执行
│
├─→ 帮人发现问题？
│   └─→ 模式3：AI引导发现
│
├─→ 即时响应用户问题？
│   └─→ 模式4：AI实时反馈
│
├─→ 提前识别风险？
│   └─→ 模式5：AI预测预警
│
└─→ 帮人找到信息？
    └─→ 模式6：AI知识检索
```

### 模式组合

复杂的教育AI产品通常是多种模式的组合：

| 产品类型 | 主模式 | 辅模式 | 说明 |
|----------|--------|--------|------|
| 教学质量诊断平台 | 模式1 | 模式3+5 | 辅助决策+引导发现+预警 |
| AI数字教师 | 模式4 | 模式6 | 实时反馈+知识检索 |
| 智能作业系统 | 模式2 | 模式4 | 自动批改+即时反馈 |
| 学习预警平台 | 模式5 | 模式1 | 预警+辅助决策 |

## 通用设计原则

无论哪种模式，都需要遵循以下原则：

### 1. 权限清晰
- AI能做什么、不能做什么，必须明确界定
- 关键决策（如给学生打不合格）必须有人的参与

### 2. 反馈闭环
- 人工干预的结果要回流给AI
- 形成"AI建议 → 人工修正 → AI学习"的循环

### 3. 透明度
- 用户（教师/学生/管理者）需要知道"这是AI的建议"
- AI的置信度和局限性必须展示

### 4. 渐进式信任
- 新AI功能先以"辅助"角色引入
- 随着准确率提升和数据积累，逐步扩展权限
- 不要一上来就让AI做最终决策

### 5. 退出机制
- 任何AI功能都可以被关闭
- AI故障时系统能降级到人工模式
