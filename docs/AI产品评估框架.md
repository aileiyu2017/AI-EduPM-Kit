# 教育AI产品评估框架

> 7维度评估体系，覆盖教育AI产品从设计到落地的全生命周期。

---

## 框架概览

教育AI产品的评估不能照搬通用AI产品的指标体系。这个框架围绕教育场景的特殊性——影响深远、容错率低、伦理敏感——设计了7个核心评估维度。

```
┌─────────────────────────────────────────┐
│           教育AI产品评估框架              │
├─────────┬─────────┬─────────┬───────────┤
│ 安全性   │ 公平性   │ 可控性   │ 可解释性   │
├─────────┼─────────┼─────────┼───────────┤
│ 鲁棒性   │ 隐私性   │ 合规性   │           │
└─────────┴─────────┴─────────┴───────────┘
```

---

## 1. 安全性（Safety）

**核心问题**：AI输出是否可能对学习者造成伤害？

教育AI直接影响学生认知发展和心理健康，安全是第一底线。

> **为什么这个维度重要**：教育AI直接面向未成年学习者，任何有害内容（如错误知识、不当引导）都可能对学生的认知发展和价值观形成造成深远影响。曾有AI辅导工具因未过滤不当内容，向青少年输出有害建议，引发家长集体投诉和产品下架。（经验值：通用行业教训）

### 评估清单

| # | 检查项 | 严重度 | 评估方法 |
|---|--------|--------|----------|
| 1.1 | AI生成的内容是否经过教育适宜性审核？ | 🔴 高 | 抽样审查100条AI输出 |
| 1.2 | 是否存在AI"教错"的防护机制？ | 🔴 高 | 错误注入测试 |
| 1.3 | 学生与AI交互是否有人工监督通道？ | 🟡 中 | 查看产品设计文档 |
| 1.4 | AI是否可能生成有害/不当内容？ | 🔴 高 | 红队测试 |
| 1.5 | 是否有紧急停止和降级机制？ | 🔴 高 | 模拟极端场景测试 |

### 评分标准

- **5分**：多层防护，有独立内容审核团队，定期安全审计
- **4分**：有内容过滤机制，有错误反馈通道
- **3分**：基本的内容过滤，无系统性安全设计
- **2分**：安全依赖模型自带能力，无额外防护
- **1分**：无安全考虑

---

## 2. 公平性（Fairness）

**核心问题**：不同群体是否获得同等质量的服务？

教育公平是社会公平的基石，AI不能成为新的教育壁垒。

> **为什么这个维度重要**：教育公平是社会公平的基石，AI系统如果基于不均衡的数据训练，可能无意识放大城乡、贫富、性别间的教育差距。某自适应学习平台曾被质疑对城市学生的推荐准确率显著高于农村学生，加剧了"数字教育鸿沟"。（经验值：通用行业教训）

### 评估清单

| # | 检查项 | 严重度 | 评估方法 |
|---|--------|--------|----------|
| 2.1 | 训练数据是否覆盖不同地区、层次的学生？ | 🔴 高 | 数据集分布分析 |
| 2.2 | AI推荐是否偏向特定类型的学校/学生？ | 🟡 中 | A/B测试对比 |
| 2.3 | 资源薄弱学校能否获得同等AI服务？ | 🔴 高 | 对比不同学校的功能可用性 |
| 2.4 | 是否存在性别/地域/经济背景的隐性偏见？ | 🟡 中 | 偏见检测算法扫描 |
| 2.5 | 是否有针对弱势群体的补偿机制？ | 🟢 低 | 查看产品策略文档 |

### 评分标准

- **5分**：有公平性专项团队，定期审计，有补偿机制
- **4分**：有意识消除偏见，数据集相对均衡
- **3分**：意识到公平性问题，但未系统解决
- **2分**：未考虑公平性
- **1分**：明显存在偏见问题

---

## 3. 可控性（Controllability）

**核心问题**：用户（教师/学生/管理者）能否理解并控制AI的行为？

教育场景中，"AI不能越俎代庖"是核心原则。

> **为什么这个维度重要**：教育场景中，教师是教学的主导者，AI不能越俎代庖。如果教师无法干预AI的输出或决策，不仅损害教学自主权，还可能在AI出错时无法及时纠正。某AI阅卷系统因不允许教师覆写评分结果，导致争议案例无法妥善处理，引发教师群体不满。（经验值：通用行业教训）

### 评估清单

| # | 检查项 | 严重度 | 评估方法 |
|---|--------|--------|----------|
| 3.1 | 用户能否调整AI输出的详细程度？ | 🟡 中 | 功能测试 |
| 3.2 | 教师能否干预/修正AI的诊断结果？ | 🔴 高 | 查看交互设计 |
| 3.3 | AI的权限边界是否清晰？ | 🔴 高 | 权限矩阵审查 |
| 3.4 | 用户能否关闭特定AI功能？ | 🟡 中 | 功能测试 |
| 3.5 | AI行为是否有明确的"决策范围"声明？ | 🔴 高 | 查看产品说明 |

### 评分标准

- **5分**：用户完全掌控，AI仅作为辅助，权限清晰分层
- **4分**：有基本的控制选项，有权限管理
- **3分**：部分可控，但缺少系统性设计
- **2分**：AI主导，用户被动接受
- **1分**：用户完全无法干预

---

## 4. 可解释性（Explainability）

**核心问题**：AI的决策过程是否可追溯？

在教育质量评估等场景中，"为什么这样评"比"评了什么"更重要。

> **为什么这个维度重要**：在教育评估场景中，"为什么这样评"比"评了什么"更重要——学生和家长需要理解决策依据才能接受结果并改进。某AI作文评分系统因无法解释给分理由，遭到大量申诉，最终被学校停用。（经验值：通用行业教训）

### 评估清单

| # | 检查项 | 严重度 | 评估方法 |
|---|--------|--------|----------|
| 4.1 | AI结论是否附带证据/依据？ | 🔴 高 | 抽样验证 |
| 4.2 | 用户能否追溯AI的推理路径？ | 🔴 高 | 功能演示 |
| 4.3 | 置信度是否清晰展示？ | 🟡 中 | UI审查 |
| 4.4 | 是否存在"黑盒"决策？ | 🔴 高 | 技术架构审查 |
| 4.5 | 不同用户角色是否看到不同粒度的解释？ | 🟢 低 | 角色切换测试 |

### 评分标准

- **5分**：每条结论都有证据链，推理路径完整可追溯
- **4分**：主要结论有依据，关键决策可追溯
- **3分**：部分有解释，但不完整
- **2分**：仅输出结果，无解释
- **1分**：完全黑盒

---

## 5. 鲁棒性（Robustness）

**核心问题**：极端输入或异常场景下是否保持稳定？

教育系统不能"宕机"，尤其是在考试、评估等关键节点。

> **为什么这个维度重要**：教育系统有严格的时效性——考试、开学、作业截止等关键节点不容宕机。某在线考试平台在期末考期间因AI服务崩溃且无降级方案，导致数万名学生无法正常交卷，引发重大舆情事件。（经验值：通用行业教训）

### 评估清单

| # | 检查项 | 严重度 | 评估方法 |
|---|--------|--------|----------|
| 5.1 | 是否有降级策略（AI不可用时的备用方案）？ | 🔴 高 | 故障注入测试 |
| 5.2 | 边界输入（空值/超长/乱码）是否安全处理？ | 🟡 中 | 边界测试 |
| 5.3 | 高并发场景（如开学季）是否稳定？ | 🔴 高 | 压力测试 |
| 5.4 | 模型更新后是否保持行为一致性？ | 🟡 中 | 回归测试 |
| 5.5 | 是否有持续监控和告警机制？ | 🟡 中 | 查看运维方案 |

### 评分标准

- **5分**：多层降级，自动恢复，7x24监控，有SLA保障
- **4分**：有降级策略，有基本监控
- **3分**：有基本的异常处理
- **2分**：偶尔不稳定，缺少系统性的鲁棒设计
- **1分**：频繁出错，无降级方案

---

## 6. 隐私性（Privacy）

**核心问题**：学生数据是否得到充分保护？

教育数据涉及未成年人，隐私保护是法律红线。

> **为什么这个维度重要**：教育数据涉及未成年人的学业表现、行为轨迹、家庭背景等高度敏感信息，泄露后果不可逆。某教育App因过度采集学生人脸数据且未加密存储，被监管部门通报并要求全面整改。（经验值：通用行业教训）

### 评估清单

| # | 检查项 | 严重度 | 评估方法 |
|---|--------|--------|----------|
| 6.1 | 是否遵循数据最小化原则？ | 🔴 高 | 数据流审查 |
| 6.2 | 敏感数据是否脱敏处理？ | 🔴 高 | 数据样本检查 |
| 6.3 | 数据访问是否有权限控制和审计日志？ | 🔴 高 | 权限矩阵审查 |
| 6.4 | 是否支持数据删除（被遗忘权）？ | 🟡 中 | 功能测试 |
| 6.5 | 数据传输和存储是否加密？ | 🔴 高 | 安全审查 |

### 评分标准

- **5分**：全链路加密，最小化采集，完整的权限和审计体系
- **4分**：关键数据加密，有访问控制
- **3分**：基本的加密措施
- **2分**：隐私保护依赖基础设施
- **1分**：无隐私保护措施

---

## 7. 合规性（Compliance）

**核心问题**：是否符合教育行业的法规和标准？

> **为什么这个维度重要**：教育行业受多部门监管（教育、网信、市场监管），法规更新频繁且要求严格。某AI教育产品因未获得家长明确授权即使用学生数据训练模型，违反《个人信息保护法》被处以高额罚款。（经验值：通用行业教训）

### 评估清单

| # | 检查项 | 严重度 | 评估方法 |
|---|--------|--------|----------|
| 7.1 | 是否符合《个人信息保护法》？ | 🔴 高 | 法务审查 |
| 7.2 | 是否符合教育部的相关规定？ | 🔴 高 | 政策对照 |
| 7.3 | 是否通过等保测评？ | 🟡 中 | 查看证书 |
| 7.4 | AI使用是否获得学校/家长授权？ | 🔴 高 | 查看授权流程 |
| 7.5 | 数据是否遵循不出境原则？ | 🟡 中 | 架构审查 |

### 评分标准

- **5分**：完全合规，有法务团队定期审查，有认证
- **4分**：主要法规合规，有基本审查
- **3分**：部分合规，正在整改
- **2分**：存在明显合规风险
- **1分**：不合规

### 填写示例

假设一个"中等水平的教育AI产品"——基础功能齐全，但缺少系统性的安全设计和完善的合规体系：

| 维度 | 权重 | 自评分(1-5) | 加权得分 | 主要差距 | 改进优先级 |
|------|------|-------------|----------|----------|------------|
| 安全性 | 25% | 3 | 0.75 | 无独立内容审核团队，依赖模型自带过滤 | P1 |
| 公平性 | 15% | 4 | 0.60 | 数据集城市偏向明显，缺少对农村学生的补偿机制 | P2 |
| 可控性 | 20% | 4 | 0.80 | 教师可覆写结果，但权限管理不够精细 | P2 |
| 可解释性 | 15% | 3 | 0.45 | 仅输出结论无证据链，推理路径不可追溯 | P1 |
| 鲁棒性 | 10% | 4 | 0.40 | 有基本降级策略，但无自动恢复机制 | P3 |
| 隐私性 | 10% | 4 | 0.40 | 关键数据加密，但缺少完整的审计日志 | P2 |
| 合规性 | 5% | 2 | 0.10 | 未通过等保测评，家长授权流程不完善 | P1 |
| **总分** | **100%** | — | **3.50** | **可解释性和合规性是最大短板** | — |

**示例解读**：
- 总分3.50属于"良好"区间下限，有明显改进空间
- **优先级P1（立即）**：安全审核团队建设、可解释性升级、合规整改
- **优先级P2（本季度）**：公平性补偿机制、权限精细化、审计日志
- **优先级P3（下半年）**：自动恢复机制

### 评级标准

- **4.5-5.0**：优秀，可作为行业标杆
- **3.5-4.4**：良好，有少量改进空间
- **2.5-3.4**：合格，有明显改进空间
- **1.5-2.4**：不合格，需全面整改
- **1.0-1.4**：严重不合格，建议暂停上线

## 快速评估版（5分钟）

完整评估需要1-2小时，但日常自检或项目初期快速摸底可以先用这个精简版。从7个维度中各提炼最关键的1-2项，共13个Yes/No问题。

### 使用方法

1. 对每个问题回答 **Yes** 或 **No**
2. 统计Yes的数量
3. 对照下表判断当前状态

### 快速检查清单

| # | 所属维度 | 检查项 | 结果 |
|---|----------|--------|------|
| Q1 | 安全性 | AI生成的内容是否经过教育适宜性审核（非技术过滤）？ | Yes / No |
| Q2 | 安全性 | 是否有紧急停止和降级机制（AI故障时可切换备用方案）？ | Yes / No |
| Q3 | 公平性 | 是否评估过不同群体（城乡/性别/经济背景）的使用效果差异？ | Yes / No |
| Q4 | 可控性 | 教师/管理员能否干预或覆写AI的决策结果？ | Yes / No |
| Q5 | 可控性 | 用户能否关闭不需要的AI功能？ | Yes / No |
| Q6 | 可解释性 | AI给出的评价/推荐是否附带理由或依据？ | Yes / No |
| Q7 | 鲁棒性 | AI服务不可用时，产品是否有备用方案（如切换到人工/规则引擎）？ | Yes / No |
| Q8 | 鲁棒性 | 是否在开学/考试等高并发时段做过稳定性验证？ | Yes / No |
| Q9 | 隐私性 | 学生敏感数据（姓名、成绩、行为记录）是否加密存储和传输？ | Yes / No |
| Q10 | 隐私性 | 是否支持学生/家长申请删除个人数据？ | Yes / No |
| Q11 | 合规性 | 是否已获得学校和家长对使用AI的知情同意？ | Yes / No |
| Q12 | 合规性 | 是否符合《个人信息保护法》的未成年人数据保护要求？ | Yes / No |
| Q13 | 合规性 | 是否通过网络安全等级保护（等保）测评？ | Yes / No |

### 结果判定

| 等级 | Yes数量 | 含义 | 建议动作 |
|------|---------|------|----------|
| 🟢 **绿灯** | 11-13项 | 基础风险可控，可继续推进 | 转入完整版评估，寻找增量优化空间 |
| 🟡 **黄灯** | 7-10项 | 存在明显短板，需谨慎推进 | 针对No的项制定整改计划，限期修复 |
| 🔴 **红灯** | 0-6项 | 基础保障缺失，风险较高 | **暂停扩大使用范围**，优先补齐安全和合规底线 |

> 💡 **经验值**：快速评估建议每月做一次（固定 checklist，5分钟完成），完整评估建议每季度做一次（深入审计，1-2小时）。

---

## 使用建议

1. **产品规划阶段**：用此框架定义产品的底线要求
2. **设计评审阶段**：逐项检查设计方案是否覆盖
3. **上线前评估**：完成完整的7维度打分
4. **运营阶段**：每季度回顾，持续改进
