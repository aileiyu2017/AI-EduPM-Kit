## 为什么需要这个工具集

教育AI不是通用AI的简单搬运。它面临独特的三重约束：

- **伦理约束**：教育影响人的成长，AI不能"试错"
- **效果约束**：教育效果滞后且难量化，无法像推荐系统那样用CTR衡量
- **场景约束**：课堂、教务、评估、辅导——每个场景对AI的容错率不同

这个工具集是我（[@aileiyu2017](https://github.com/aileiyu2017)）在高校教学质量诊断、AI数字教师、政务数据洞察等项目中沉淀的方法论，抽象为可复用的框架和模板。

## 工具箱内容

### 1. AI产品评估框架 [→ 查看](docs/assessment-framework.md)

覆盖教育AI产品的7个评估维度：

| 维度 | 核心问题 | 评估要点 |
|------|----------|----------|
| **安全性** | AI输出是否可能对学习者造成伤害？ | 内容过滤、异常检测、人工兜底机制 |
| **公平性** | 不同群体是否获得同等质量的服务？ | 偏见检测、资源均衡、弱势群体覆盖 |
| **可控性** | 用户能否理解并控制AI的行为？ | 透明度设计、参数可调、关闭选项 |
| **可解释性** | AI的决策过程是否可追溯？ | 证据链、推理路径、置信度展示 |
| **鲁棒性** | 极端输入下是否保持稳定？ | 边界测试、降级策略、异常处理 |
| **隐私性** | 学生数据是否得到充分保护？ | 数据最小化、脱敏处理、访问控制 |
| **合规性** | 是否符合教育行业法规？ | 等保、个保法、教育部相关规定 |

### 2. RAG应用设计 Checklist [→ 查看](docs/rag-checklist.md)

从知识库构建到问答交互的完整设计清单，覆盖：
- 文档解析与结构化策略
- 向量化与检索方案选型
- 上下文组装与prompt工程
- 答案可信度设计（引用/置信度/兜底）
- 幻觉检测与防控机制

### 3. 人机协同设计模式 [→ 查看](docs/human-ai-patterns.md)

教育场景中6种典型的人机协作模式：

```
AI辅助决策 → AI生成建议 + 人工审核确认
AI增强执行 → AI处理重复任务 + 人工聚焦核心判断
AI引导发现 → AI提出问题线索 + 人工深入探究
AI实时反馈 → AI即时响应 + 人工长期跟踪
AI预测预警 → AI风险识别 + 人工干预处置
AI知识检索 → AI信息召回 + 人工综合判断
```

## 5分钟快速开始：一个完整的评估示例

**场景**：你正在设计一个"AI教学评估系统"——让老师上传课件和课堂录音，AI自动生成教学诊断报告。在产品评审前，你想用评估框架做一次自检。

### Step 1：打开评估框架（30秒）

打开 [`docs/assessment-framework.md`](docs/assessment-framework.md)，找到评分表。你会看到7个维度，每个维度1-5分，总分35分。

### Step 2：逐项评分（3分钟）

根据你当前产品的状态，诚实地打分。以下是**一个中等成熟度产品的示例评分**（供你参考打分尺度）：

| 维度 | 得分 | 判断依据（示例） |
|------|:----:|----------------|
| **安全性** | 4 | 已部署内容过滤，拦截明显有害输出；但异常检测仅覆盖10%的边界场景 |
| **公平性** | 3 | 未做显式的偏见检测；不同学科的诊断标准可能隐含偏差 |
| **可控性** | 4 | 老师可调整诊断维度的权重；但AI生成报告后无法逐条修改结论 |
| **可解释性** | 4 | 每条诊断建议附带引用来源（课件/录音片段）；但推理链未可视化展示 |
| **鲁棒性** | 3 | 有基本的降级策略（超时返回兜底文案）；边界测试覆盖不足 |
| **隐私性** | 5 | 数据最小化采集，录音实时转写后删除原始文件，符合等保2.0 |
| **合规性** | 4 | 已完成个保法合规审查；教育部AI教育应用管理办法尚未完全对标 |

**总分：27/35（77%）**

### Step 3：定位优先改进项（1分钟）

查看评分最低的维度：

- **公平性（3分）** → 优先行动：在诊断报告中增加"群体差异分析"，对比不同学科/学段教师的诊断得分分布，发现潜在偏差
- **鲁棒性（3分）** → 优先行动：补充边界测试用例（空课件、无声音频、超长文档），定义明确的降级策略

### Step 4：记录并推进

把评分表复制到你的产品文档中，作为本次评审的记录。针对2个优先改进项，创建开发任务并指派负责人。

> ✅ **完成。** 你已经在5分钟内完成了一次系统性的产品自检，并带回了2个可执行的改进项。
>
> 下一步：针对系统的RAG功能（课件解析/知识召回），用 [`docs/rag-checklist.md`](docs/rag-checklist.md) 做第二轮专项检查。
